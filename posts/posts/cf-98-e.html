题目大意.

现在有一副牌，共 n+m+1 张，牌之间两两不同。先手发到了 n 张，后手发到了 m 张，还剩一张弃置。双方都不知道对手的手牌和弃置的牌。

先后手轮流行动，每一回合都可以选择如下两种操作的一种：

  • 猜测弃置的牌是哪张。若猜中则己方胜利；否则对手胜利。不论如何，游戏立即结束。
  • 指定一张牌，对手必须如实回答这张牌是否在自己手上。

求：采取最佳策略时，先手的胜率。

n,m≤1000。

正常的玩法就是依次问不在自己手中的牌，然而这个游戏不正常。

询问者可以选择

  • 真的在问；
  • 询问一张在自己手中的牌，以迷惑对手。

对手也可以选择

  • 认为对方真的在问；
  • 认为对方在骗自己。

下面我们对 2×2 种情形进行分析。

1. 真的在问

有 mm+1 的可能性，询问者问了一张在受询者手中的牌，显然他是真的在问。现在双方都知道了这张牌在受询者手中，相当于从他手里弃掉了这张牌；

有 mm+1 的可能性，这张牌就是弃置的牌，询问者也得知了这一信息。

  • 如果受询者相信对方真的在问，那他也就相信这张牌是弃置牌，从而在下回合会报出答案并胜利。
  • 如果受询者相信对方在骗自己，那么他会把这张牌从待选答案中删除，他永远也猜不对了，而询问者会在下下回合胜利。

2. 欺骗

  • 如果受询者相信对方真的在问，那他也就相信这张牌是弃置牌，从而在下回合会报出答案并输掉。
  • 如果受询者相信对方在骗自己，那么他会把这张牌从待选答案中删除，相当于从对方手里弃掉了这张牌。

接下来只需要求解这个问题的纳什均衡。设先手的询问概率为 p，显然，在均衡条件下，后手不管相信先手在问还是相信先手在欺骗都不能占到便宜，于是不难列出方程解出 p，也就得出了策略。

#include<bits/stdc++.h>
using namespace std;
typedef double db;


bool vis[1005][1005];
db f_[1005][1005];


db f(int n, int m) {
    if (vis[n][m]) return f_[n][m];
    db X = 1.0 / (m + 1);
    if (!n) return X;
    if (!m) return 1;
    vis[n][m] = 1;
    db A = X * m * (1 - f(m - 1, n)), B = X + A;
    db C = 1, D = 1.0 - f(m, n - 1);
    db P = (D - C) / (A - C - B + D);
    return f_[n][m] = P * A + (1 - P) * C;
}


int main() {
    int n, m; scanf("%d%d", &n, &m);
    db ans = f(n, m);
    printf("%.10lf %.10lf\n", ans, 1 - ans);
}
