本篇文章以树上的最大点权独立集为例，讲解一下DDP。

如果不带修，那么就是P1352 没有上司的舞会，设f[0][i]表示不选i节点时，i的子树获得的最大权值，f[1][i]表示选i时的最大权值，于是轻松写出转移方程：

f[0][i]=∑j是i的子节点max(f[0][j],f[1][j])

f[1][i]=A[i]+∑j是i的子节点f[0][j]

然后O(N)愉快AC。

为了支持修改，我们先引入一种新的矩阵乘法。

C=A×B表示C[i][j]=max(A[i][k]+B[k][j])

不太显然地，它满足分配律，即(A×B)×C=A×(B×C)。这将是我们用线段树维护它的重要根据。

讨论一下简单情况：树退化成一条链。除了一个叶子节点每个节点都只有一个儿子。从根到叶子依次编号1⋅⋅⋅N。

于是之前的转移可以写成矩阵乘法的形式：

[f[0][i]f[1][i]]=[00A[i]−∞]×[f[0][i+1]f[1][i+1]]

可以自己手算验证。

同样，

[f[0][i]f[1][i]]=[00A[i]−∞]×[00A[i+1]−∞]×...×[00A[N]−∞]×[00]

考虑到这种新的矩阵乘法满足分配率，也就是满足区间加（乘？）法，左区间×右区间=整个大区间，所以，我们可以用线段树维护！每次修改一个点的权值就是在修改对应位置的矩阵的左下元素，最后调用整条链上所有矩阵的积再乘上[00]即可。

然而，在树上要比在链上复杂很多，因为一个点可能有很多很多儿子。我们思考，能否把一棵树变成一条链呢？可以的，比如树链剖分就可以，所以我们下面讲解树剖做法。（因为我不会LCT）

首先，我们还是采用矩阵思想。如果改变了一个节点的权值，那么从它而上一直到根节点的点会受影响而其他点不会。既然我们已经树链剖分了，那么就利用一下：不断跳到所在重链顶端（的父亲节点），然后修改跳过的这条重链（具体怎么修改，马上就讲）。

注意到我们除了重链间分隔的轻边，我们只会跳过重链，所以设计一个i所在重链被怎么瞎折腾都不会变的东西是极好的，我们设g[0][i]和g[1][i]，它和f[0][i],f[1][i]的唯一区别就是，我们算g[0][i]和g[1][i]的时候我们假设i的重儿子和它的子树不存在，即：

g[0][i]=∑j是i的轻儿子节点max(f[0][j],f[1][j])

g[1][i]=A[i]+∑j是i的轻儿子节点f[0][j]

（请注意，i的轻儿子节点的重儿子还是被算进转移里的，所以它继承的是f[k][j]）

如果我们已经知道了g[0][i]和g[1][i]，f[k][i]会非常好求，也就是再重新把重儿子考虑进去即可：

f[0][i]=g[0][i]+max(f[0][son[i]],f[1][son[i]])

f[1][i]=g[1][i]+f[0][son[i]]

（son[i]是i的重儿子）

注意，它也可以写成矩阵乘法！

[f[0][i]f[1][i]]=[g[0][i]g[0][i]g[1][i]−∞]×[f[0][i+1]f[1][i+1]]

于是我们现在用线段树维护[g[0][i]g[0][i]g[1][i]−∞]这个矩阵。（然而，在代码里写的时候右下角是0，事实证明两种在此题都可以，原因留读者思考）

所以，要修改u的权值为x的时候，我们就要修改u的矩阵。g[0][u]显然完全没有被影响到，但是我们看到g[1][u]的转移方程里有A[i]，所以它会被影响，影响的数值就是−A[i]+x。然后不要忘了把x赋给A[i]。然而，修改u会有一系列连锁反应，我们下面进行讨论。

这里mark一下

我们在跳重链时，比如从u跳到fa[top[u]]（top[u]是u所在重链的顶端）时：

  • u→top[u]的这条链是重链，它的g不需要改（我们设计g可就是为了这个啊）；

  • top[u]→fa[top[u]]是一条轻边（否则top[u]上面还有重边，那这条重链不就可以继续往上延伸了？），fa[top[u]]的矩阵需要修改。

修改多少呢？就应该是top[u]所在整条重链的所有矩阵之积（线段树上一个Query搞定）（因为是差值所以其实要两个）在u被改掉之后的差值（这里的矩阵的差还是和正常一样的，对齐减一减即可）。

（上面这段可能较难理解，建议再回去看一下转移方程）

注意，这时fa[top[u]]的矩阵是不是被修改了？它就应该作为新的u，继续进行上面的步骤！（我们可没有修改fa[top[u]]的权值，所以从mark的地方继续即可）

至此，P4719完成。

    mATRIx(){a[0][0]=a[0][1]=a[1][0]=a[1][1]=0;}

        for(int j=0;j<2;j++)
        for(int k=0;k<2;k++)

        return c;
    }

void DFS1(int u,int f){
    F[1][u]=max(0,A[u]);
    fa[u]=f;siz[u]=1;dep[u]=dep[f]+1;
    for(int e=lnk[u];e;e=pre[e])if(tgt[e]!=f){

    }

    for(int e=lnk[u];e;e=pre[e])if(tgt[e]!=f&&tgt[e]!=son[u]){

void Build(int x,int l,int r){

            g1+=F[0][tgt[e]];

    }

void Update(int x,int l,int r,int pos){

    else Update(x*2+1,mid+1,r,pos);

}
mATRIx Query(int x,int l,int r,int L,int R){
    if(L<=l&&r<=R) return sum[x];
    int mid=(l+r)>>1;
    if(L>mid) return Query(x*2+1,mid+1,r,L,R);

    return Query(x*2,l,mid,L,R)*Query(x*2+1,mid+1,r,L,R);

例题讲解：SP1716 GSS3 - Can you answer these queries III

开讲前顺便提一句，GSS这个系列里面全是毒瘤维护题，有兴趣可以做一下。其中有些题目还有一些很乱来但很有趣的做法，比如这题的DDP。

众所周知此题正解是线段树，但是……

我们先看询问，这是一个你刚学DP就知道的经典问题，DP+修改，离DDP只有一步：搞出矩阵了。

容易写出：f[i]表示必选i作为末尾时的最长子段，g[i]则是仅考虑1⋯i时的答案（你总不可能询问的时候重新扫一遍吧，所以必须把这个带着一起转移）。那么转移方程如下：

f[i]=max(f[i−1],0)+A[i]

g[i]=max(g[i−1],f[i])

注意到f[i]=max(f[i−1],0)+A[i]，说明，如果我们的右矩阵（被转移的那个，左矩阵可以认为是转移关系）只有f[i],g[i]是不可能的，否则这个0不可能蹦出来。（可以思考一下）我们需要再添一个常数，比如0。

此外，g[i]也很难搞，因为它关系到f[i]。但是你可以把g[i]的方程里的f[i]给拆掉，即：

g[i]=max(g[i−1],f[i−1]+A[i],A[i])

然后就很好凑了。

小结

以我的理解，DDP其实大概就是在一条链上用线段树维护“转移”（以矩阵的形式），所以，如果你的DP化不成链又带修改……还是自闭吧。而且，矩阵的形式决定了它没法容纳某些条件判断，所以，比如说LIS带修，自闭吧。
