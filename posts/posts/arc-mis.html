ARC123F - Insert Addition

项目等级：Safe

题意等价于，先 f∞ 再删去大于 N 的元素、

莫名想到 SBT？即 (A,B) 能生成的恰是 xA+yB(gcd(x,y)=1)。

考虑 (A,B) 能生成多少元素。有多少 gcd(x,y)=1 的 (x,y) 满足 xA+yB≤N？

忽略 gcd(x,y) 是好做的，加上它时我们需要一个容斥，即要对一排 N 算忽略 gcd(x,y) 时的答案。注意相邻两项相差的恰好是 xA+yB=N，于是可以 O(1) 算。因此原问题就可以在 O(N) 时间内算出。

然而在 SBT 上直接走可能需要走 O(N) 层，无法接受。

当 B≥N，合法的 y 非常有限，爆枚之；问题就变成有多少 x≤X 与 y 互质，还是可以爆枚预处理。于是时间复杂度就变为 O(NN)。

ARC125F - Tree Degree Subset Sum

项目等级：Euclid

可见要对所有 X 求和这一点非常重要，特定的 X 的贡献是非常难求出来的。

可见树这一条件非常重要。

  • 我们有很多这样的子集：大小为 x，度数和为 2x−1。即一个子树。
  • 事实上最终方案一定是许多子树的异或。
  • 度数和即为 2X−正子树数量+负子树数量。

或许 −正子树数量+负子树数量 是本质的因素，我们枚举它然后对所有 X 求和？固定 −正子树数量+负子树数量，有多少不同的 X？奇怪的问题……

爬去看题解了

不妨先把度数全 −1。Y 也相应地变为 Y−X。

我们断言：【对于特定的 Y，合法的 X 是一段区间】。

考虑使用 X 最多的方案和最少的方案所用的 X 数目之差；只需要证明这个值不超过 0（叶子）数的两倍。而这是因为，不难发现，−正子树数量+负子树数量 的绝对值不会超过叶子数。

然后因为 d 只有 n 种，于是就有 O(nn) DP 了。

ARC126F - Affine Sort

项目等级：Safe

一看题就知道是 ATC 特有的神秘数论……

记 f(K)=∑i=1Kg(i)（其实 g(K) 就是 c 取 K 时的答案）。

（下面这步推导完全出于实用主义考量，毫无严谨性，请不要参考！）

limK→∞f(K)K3=limK→∞f′(K)3K2(洛必达法则)≈13limK→∞g(K)K2

（UPD 2021/10/30：那个离散形式的洛必达是 Stolz－Cesàro theorem —— serverkiller）

考虑 g(K)。把 y=(Ax+B)modK 看成是一个在环上以速度 x 旋转的点，当所有点以 y1,y2,…yn 的顺序排列整齐时旧日支配者就会苏醒就会计入答案。

  • 不需要太关心 B；当 y1,y2,…,yn 在环上顺时针排列时，合法的 B 的数量就是 (y1−yn)modK。

自然考虑，什么时候 yi 会超越 yi+1 或 y1（或被它们超越）。相对速度为 |xi−xi+1|，故次数为 K/(K/|xi−xi+1|)=|xi−xi+1|。

数据范围里已经写了，∑xi≤5×105，因此我们可以直接跟踪所有点的相对位置。于是剩下的工作已经很显然了：合法的 B 就是数个梯形的面积。

ARC127F - ±AB

项目等级：Euclid

如果 A+B≤M 那我们可以直接报告 M+1（弱周期引理）。

那么 A,B 中的较大者（不妨设为 B）必定大于等于 M/2。说不定可以暴力做某事？

重复以下步骤：

  • 记录所有 X±kA；（gcd(A,B)=1，总是不会记重）
  • 令 X 为其中的最大值 −B。（同理再做一遍最小值 +B）

如果 A 很小，相位变化（BmodA）也会很小。无法接受。

踩到特定的一段相位会去世（不能继续 −B）。能继续规约，变成辗转相除吗？

理一下思路。考虑那个 AGC045F 的 trick（讲道理 maroonrk 是不是用同一个 idea 出了这两道题……）。

把 X 不断自加 A 的过程看成光线。当 X>M−A 时会去世。考虑把相位的变动看成新的光线，继续解决这个子问题就能知道去世时间了。

ARC128F - Game against Robot

项目等级：Keter

首先有一个简单的策略：Snuke 是不是直接贪心地烧就可以了呢？显然不是，见样例 2。

即，我们会因为一张卡快被烧了而救它，会因为一张卡很晚被烧就很晚才吃。

考虑 p=n 和 p=n−1 的两张卡，这两张卡我们最多留一张，当然也可以一张都不留用来吃后面的卡。

怎么判断什么时候留下时间吃后来的卡？

考虑从后往前扫，我们有两种选择：

  • 吃掉当前两张卡的一张；
  • 救一张曾经被我们抛弃的卡。

其实可以进一步简化：把当前的两张卡 push 进堆中，然后吃掉堆顶。

（其实用网络流模型理解也可以）

那么现在的任务是把这个过程翻译为 DP……不对啊 n=106 咋 DP。

考虑拆贡献？有多少排列能让一张特定的卡被吃？所有排列中，有多少张 ≥X 的卡被吃？

这时，所有元素和排列的内容全都退化为了 0/1：≥X 还是 <X。（当然最终算排列数的时候要乘 (X−1)!(N−X+1)!）

考虑从前往后扫时 0 数量的前缀和，第 i 步时我们已经遇到了 Si 个 0。我们断言，没被吃的 0 数量恰为 max(Si−i)，这应当是显然的。

现在把 0/1 变为 -0.5/0.5，那么没被吃的 0 数量恰为 max⌊Si⌋=⌊(maxSi)⌋。

枚举 Y，考虑有多少 maxSi≤Y。使用反射容斥，组合数减一减。最后大约是个组合数斜线和的形式，就做完了。
