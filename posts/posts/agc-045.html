项目标题：A - Xor Battle

项目等级：Safe

威胁等级：绿色

描述：

现在有两个玩家，有一个初始为 0 的变量 X。游戏进行 n 回合，第 i 回合时：

  • 玩家 si 行动，他可以选择令 X←X⊕Ai 或跳过回合。

0 号玩家的目标是使 X 最终为 0，1 号玩家的目标与之相反。

问游戏结果如何。

n≤200,Ai≤1018。100 组测试数据。

特殊收容措施：

考虑从结果倒推。

  • 每当有一个 0 回合，那么【0 胜空间】S 就会变为 S∪(S⊕A)。

  • 每当有一个 1 回合，那么 S←S∩(S⊕A)。

    • 这个操作比较神秘，但是也很好处理：如果 A∉S，那么直接 S←∅，否则没有效果。

于是就很好维护了。

#include<bits/stdc++.h>
using namespace std;
typedef long long ll;


int n;
int s[205]; ll A[205];
ll B[64];


void solve() {
    scanf("%d", &n);
    for (int i = 1; i <= n; i++) scanf("%lld", &A[i]);
    for (int i = 1; i <= n; i++) scanf("%1d", &s[i]);


    memset(B, 0, sizeof(B));


    for (int i = n; i; i--)
        if (s[i]) {
            ll X = A[i];
            for (int j = 63; j >= 0; j--) if ((X >> j) & 1)
                X ^= B[j];
            if (X) { printf("1\n"); return; }
        }
        else {
            ll X = A[i];
            for (int j = 63; j >= 0; j--) if ((X >> j) & 1) {
                if (!B[j]) B[j] = X;
                X ^= B[j];
            }
        }
    printf("0\n");
}


int main() {
    int T; scanf("%d", &T);
    while (T--) solve();
}

项目标题：B - 01 Unbalanced

项目等级：Euclid

威胁等级：绿色

描述：

现在有一个由 -1/1/? 构成的串，你要把 ? 替换成 -1/1，使得所有前缀和的极差尽量小。

n≤106。

特殊收容措施：

首先枚举最小值 X。先把所有 ? 替换成 1，如果此时仍不能使所有前缀和 ≥X 自然返回 false。

那么接下来的问题就是：看看能否把某些 ? 翻成 -1 而不是 1，以使得最大值尽量小。这里有一个策略上的问题：注意到，如果我们可以把某个 -1 往前推，我们一定会把它往前推。因此从前往后贪心地翻是正确的。

接下来我们断言，只有最大可行的 X 和它减去 1 可能成为答案。直观地说，如果令 X←X−2，那我们最多也只能多翻一次，相比原来的 X 并不优。

于是直接模拟就做完了。

xxxxxxxxxx
#include<bits/stdc++.h>
using namespace std;
typedef long long ll;


const int maxn = 1000005;


int n;
char s_[maxn];
int s[maxn];
int Ss[maxn];
int MSs[maxn];


int calc(int X) {
    int lzy = 0, ans = 0;
    for (int i = 1; i <= n; i++) {
        if (s[i] == 0 && MSs[i] + lzy - 2 >= X) lzy -= 2;
        assert(Ss[i] + lzy >= X);
        ans = max(ans, Ss[i] + lzy);
    }
    return ans - X;
}


int main() {
    scanf("%s", s_ + 1);
    n = strlen(s_ + 1);
    for (int i = 1; i <= n; i++)
        if (s_[i] == '0') s[i] = -1;
        else if (s_[i] == '1') s[i] = 1;
        else s[i] = 0;
    
    int X = 0, now = 0;
    for (int i = 1; i <= n; i++)
        now += (s[i] == -1 ? -1 : 1),
        X = min(X, now),
        Ss[i] = now;
    MSs[n] = Ss[n];
    for (int i = n - 1; i; i--)
        MSs[i] = min(Ss[i], MSs[i + 1]);
    
    int ans = min(calc(X), calc(X - 1));
    printf("%d\n", ans);
}

项目标题：C - Range Set

项目等级：Euclid

威胁等级：红色

描述：

现在有一个初始全为 0 的长度为 n 的序列。

你可以做任意多次以下操作：

  • 把连续 A 个元素变为 0；
  • 把连续 B 个元素变为 1。

问最终结果可能有多少种。

n≤5000。

特殊收容措施：

考虑一个最终结果能不能被达成。

不停重复以下操作：

  • 把一个长度 ≥A 的全白序列变成全黑。
  • 把一个长度 ≥B 的全黑序列变成全白。

直观地想，这个策略很有可能是最优的。它也确实是最优的。但我不会证。

一个全白序列翻成全黑后会和旁边的黑段合并。如果这个黑段还可以翻，那么我们已经赢了：这个序列的长度一定又 ≥A 又 ≥B，不断反转它的颜色就能吃掉其他所有的段。

可见一个不能被达成的序列的形态是极其有限的，必须在总体来说是 <A 的白色和 <B 的黑色交替，黑色中间可能套一些 ≥A 的白色（反之亦然，看 A,B 的大小关系。）。于是 DP 就立刻浮现了。

const int maxn = 5005, p = 1000000000 + 7;
struct Z {int x; Z(int x0 = 0) : x(x0) {}};
int inline check(int x) { return x >= p ? x - p : x; }
Z operator +(const Z a, const Z b) { return check(a.x + b.x); }
Z operator -(const Z a, const Z b) { return check(a.x - b.x + p); }
Z operator -(const Z a) {return check(p - a.x);}
Z operator *(const Z a, const Z b) { return 1LL * a.x * b.x % p; }
Z& operator +=(Z &a, const Z b) { return a = a + b; }
Z& operator -=(Z &a, const Z b) { return a = a - b; }
Z& operator *=(Z &a, const Z b) { return a = a * b; }
Z qpow(Z a, int k) {
    Z ans = 1;
    for (; k; a *= a, k >>= 1) if(k & 1) ans *= a;
    return ans;
}


int n, A, B;
Z f[maxn][maxn][2]; // pos, length of black, now color
Z g[maxn][maxn][2];
Z h[maxn][maxn];
 
int main() {
    scanf("%d%d%d", &n, &A, &B);
    if (A > B) swap(A, B);


    for (int i = 1; i < A; i++) f[i][0][0] = 1;
    for (int i = A; i <= n; i++) f[i][i][0] = 1;
    for (int i = 1; i < B; i++) f[i][i][1] = 1;


    for (int i = 1; i <= n; i++) {


        for (int j = 1; j <= min(i, A - 1); j++)
            f[i][0][0] += h[i - j][B - 1];


        for (int j = 1; j <= min(i, B - 1); j++) {
            if (j >= A + 1) {
                f[i][j][0] += g[i - A][i - j][1];
                f[i][j][0] -= g[i - j][i - j][1];
            }
                f[i][j][1] += g[i - 1][i - j][0];
                if (j != i) f[i][j][1] -= g[i - j - 1][i - j][0];
        }


        for (int j = 0; j <= min(i, B - 1); j++)
        for (int d = 0; d < 2; d++)
            g[i][i - j][d] = g[i - 1][i - j][d] + f[i][j][d];


        for (int j = 1; j < B; j++) h[i][j] = h[i][j - 1] + f[i][j][1];


    }
    Z ans = f[n][0][0];
    for (int i = 1; i < B; i++) ans += f[n][i][0] + f[n][i][1];

D - Lamps and Buttons

题目大意.

现在有一个 1∼n 的谜之排列 p（它在所有 1∼n 的排列中均匀随机；你一开始并不知道它）和 n 盏灯泡，其中 1∼A 是亮着的。你可以重复以下操作任意次：

  • 选择一个亮着的灯泡 i。切换 pi 的状态。

你的目标是让所有灯泡都变亮。

问，你获胜的概率有多大。

n≤107,A≤5000。

可见不能获胜当且仅当每个环都至少包含一个亮灯泡……吗？这题有一个阴间 cases，就是自环。当你试图操作一个亮灯泡却发现它是自环，你会瞬间爆炸。

很显然我们根本无从判断那些灯泡是自环，只能随机选一个灯泡操作，如果遇到自环就自认倒霉。但是随机这回事又让这个游戏的复杂性进一步上升了。

事实上，由于 p 是均匀随机的，所以我们只需要对 1∼A 依次操作过去，效果和随机选是一样的。（你也可以认为原来的随机策略是对 1∼A random_shuffle 后依次操作，最终还是回到这个策略）但这个策略就简单多了。

对于一个能获胜的排列，它的结构如下：t+1 处是首个自环，每个 A+1∼n 的元素所在的环都至少有一个 1∼t 的元素。

F - Division into Multiples

题目大意.

你有 X 个写着 A 的球、Y 个写着 B 的球，你要把这些球分组，是组内和是 C 的倍数的组尽量多。问最大组数。

A,X,B,Y,C≤109。2×104 组测试数据。

首先可以做如下处理：

  • A←AmodC,B←BmodC；
  • 如果 gcd(A,B,C)≠1，各自除以 gcd；
  • 如果此时还有 gcd(A,C)≠1，那么 B 中无此因子。A,C 各自除以 gcd，Y 亦除以 gcd 下取整。（因为今后放 B 球必定是 gcd 个一组）
  • 对 B,C 亦然。
  • 如果此时还有 gcd(A,B)≠1，那么 C 中无此因子。A,B 各自除以 gcd。（因为今后的总和一定形如 k⋅gcd⋅C）

现在，A,B,C 两两互质。这时我们可以直接考虑方程

Ax+By=0(modC)

其解为 (k,−AB−1k)（其中运算皆是 modC 下的）。
